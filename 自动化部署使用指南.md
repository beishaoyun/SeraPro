### 服务器信息获取

**如果你有服务器的 SSH 权限**:

```bash
# 查看操作系统信息
cat /etc/os-release

# 查看 SSH 端口
cat /etc/ssh/sshd_config | grep Port

# 检查 SSH 服务状态
systemctl status sshd
```

## ✅ 测试服务器连接

添加服务器后，**必须测试连接**以确保可以正常访问。

### 操作步骤

1. 在服务器列表中找到刚添加的服务器
2. 点击 **测试连接** 按钮
3. 查看测试结果

### 测试结果说明

✅ **连接成功**:
```
✅ SSH 连接成功
操作系统信息：Ubuntu 22.04.3 LTS
```

❌ **连接失败**:
```
❌ Failed to connect to 192.168.1.100:22
```

### 连接失败排查

**错误 1: Connection refused**
```
原因：SSH 服务未运行或端口错误
解决:
  1. 检查 SSH 服务：systemctl status sshd
  2. 重启 SSH 服务：systemctl restart sshd
  3. 确认端口：netstat -tlnp | grep ssh
```

**错误 2: Connection timed out**
```
原因：防火墙阻止或 IP 不可达
解决:
  1. 检查 IP 是否正确
  2. 检查防火墙：ufw status 或 firewall-cmd --list-all
  3. 开放 SSH 端口：ufw allow 22/tcp
```

**错误 3: Authentication failed**
```
原因：用户名或密码错误
解决:
  1. 确认用户名正确（通常是 root）
  2. 确认密码正确
  3. 尝试手动 SSH 测试：ssh root@your-server-ip
```

## 🚀 自动化部署

### 创建部署任务

#### 步骤 1: 准备 GitHub 项目

确保你的 GitHub 项目：
- ✅ 是公开的（或配置了访问权限）
- ✅ 包含 README.md 或部署说明
- ✅ 包含 Dockerfile 或部署脚本（推荐）

#### 步骤 2: 填写部署信息

1. 点击 **部署历史** → **新建部署**

2. 选择服务器
   - 从下拉列表选择已添加的服务器

3. 输入 GitHub 项目地址
   - 格式：`https://github.com/用户名/项目名`
   - 示例：`https://github.com/facebook/react`

4. 服务类型（可选）
   - web: Web 应用
   - database: 数据库服务
   - proxy: 代理服务
   - api: API 服务
   - 留空让系统自动识别

#### 步骤 3: 开始部署

点击 **创建部署** 后，系统会：

1. **分析项目** - 读取 GitHub README 和代码结构
2. **生成部署计划** - 自动识别需要的步骤
3. **执行部署** - 通过 SSH 在服务器上执行命令
4. **实时日志** - 显示每一步的执行结果

### 部署流程示例

```
步骤 1/5: 克隆代码
✓ git clone https://github.com/example/repo.git
耗时：3.2s

步骤 2/5: 安装依赖
✓ npm install
耗时：15.4s

步骤 3/5: 构建项目
✓ npm run build
耗时：45.2s

步骤 4/5: 构建 Docker 镜像
✓ docker build -t app:latest .
耗时：120.8s

步骤 5/5: 启动容器
✓ docker run -d -p 80:80 app:latest
耗时：2.1s

✅ 部署完成！
```

### 部署失败处理

如果部署失败：

1. **查看错误日志**
   - 展开失败的部署记录
   - 查看红色标记的错误信息

2. **使用 AI 排错**
   - 点击 **AI 排错** 按钮
   - AI 会分析错误并提供解决方案
   - 可以输入具体问题与 AI 对话

3. **重试部署**
   - 修复问题后
   - 点击 **重试** 按钮重新执行

## ❓ 常见问题

### Q1: 部署时卡住不动

**可能原因**:
- 网络连接问题
- 服务器响应慢
- WebSocket 断开

**解决方法**:
1. 刷新页面重新连接 WebSocket
2. 检查服务器网络状态
3. 查看部署状态，如果已完成则无需担心

### Q2: 无法访问 GitHub

**可能原因**:
- 服务器在中国大陆，无法访问 GitHub

**解决方法**:
1. 在服务器上配置 GitHub 镜像
2. 使用 Git 代理
3. 手动上传代码到服务器

```bash
# 配置 Git 代理
git config --global http.proxy http://proxy-server:port
git config --global https.proxy https://proxy-server:port
```

### Q3: Docker 空间不足

**错误信息**: `No space left on device`

**解决方法**:
```bash
# 清理未使用的镜像
docker image prune -a

# 清理未使用的容器
docker container prune

# 清理未使用的卷
docker volume prune

# 查看磁盘使用
df -h
```

### Q4: 端口冲突

**错误信息**: `Address already in use`

**解决方法**:
```bash
# 查看端口占用
lsof -i :80

# 停止冲突的进程
kill -9 <PID>

# 或者修改部署配置使用其他端口
```
