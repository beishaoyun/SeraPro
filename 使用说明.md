## 功能一：添加服务器

1. 登录后点击左侧菜单 **服务器**
2. 点击 **添加服务器** 按钮
3. 填写服务器信息：
   - **服务器名称**: 自定义名称（如：生产服务器 1）
   - **主机 IP/域名**: 服务器的 IP 地址或域名
   - **SSH 端口**: 通常是 22
   - **SSH 用户名**: 通常是 root
   - **SSH 密码**: 服务器登录密码
   - **操作系统**: 如 ubuntu、centos 等
   - **系统版本**: 如 22.04、8 等
4. 点击 **添加** 保存

### 测试服务器连接

添加服务器后，点击服务器卡片上的 **测试连接** 按钮：
- ✅ 绿色对勾 = 连接成功
- ❌ 红色叉号 = 连接失败（检查 IP、端口、密码是否正确）

## 功能二：自动化部署

### 创建部署任务

1. 点击左侧菜单 **部署历史**
2. 点击 **新建部署** 按钮
3. 填写部署信息：
   - **选择服务器**: 从下拉列表选择已添加的服务器
   - **GitHub 项目地址**: GitHub 仓库 URL（如：https://github.com/owner/repo）
   - **服务类型** (可选): 如 web、database、proxy 等
4. 点击 **创建部署** 开始部署

### 查看部署进度

创建部署后，会自动显示实时日志视图：
- 显示每个部署步骤的状态（进行中/成功/失败）
- 显示命令输出
- 显示错误信息（如有）
- 显示每步耗时

### AI 辅助排错

如果部署失败：
1. 点击部署卡片上的 **AI 排错** 按钮
2. AI 会分析错误日志并提供解决建议
3. 可以输入具体问题与 AI 对话

## 常见问题

### 1. 测试连接失败

**可能原因**:
- SSH 服务未运行：`systemctl status sshd`
- 防火墙阻止：检查端口 22 是否开放
- 密码错误：确认 SSH 密码正确
- IP 地址错误：确认服务器 IP 可达

**解决方法**:
```bash
# 检查 SSH 服务状态
systemctl status sshd

# 重启 SSH 服务
systemctl restart sshd

# 检查防火墙
ufw status
# 或
firewall-cmd --list-all
```

### 2. 部署失败

**可能原因**:
- 服务器无法连接 GitHub
- 服务器缺少必要依赖（Git、Docker 等）
- GitHub 项目不存在或私有

**解决方法**:
1. 使用 AI 排错功能获取具体建议
2. 检查服务器日志
3. 确保服务器已安装必要工具

### 3. 忘记密码

联系管理员在管理后台重置密码：
1. 管理员登录
2. 进入 **用户管理**
3. 找到对应用户
4. 点击 **重置密码** 按钮
